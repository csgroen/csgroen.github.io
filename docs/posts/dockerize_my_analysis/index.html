Creating our docker

Now we need to create our docker file. Thereâ€™s a lot more information about it here and a cheat sheet on docker here. But hereâ€™s how I think is easiest to do for R reproducible analyses.

My R docker philosophy is that we keep code and data separate from our environment. The docker will only contain the environment to reproduce the analysis. The code and data will be mounted into the docker when it is run. This means you can still modify the code once the docker image has been built and donâ€™t need to build it again every time, as long as you didnâ€™t add any new dependencies in the new code. This will greatly simplify our lives, while still ensuring full reproducibility of results.

That being said, time to assemble the Dockerfile and build our image.

For all R projects, look no further than the Rocker project for all your base image needs. What is the best version will depend on this project, but my general direction is: if you use tidyverse anywhere, use the tidyverse image.
